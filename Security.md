# 🔐 Security Policy

## 📌 Supported Versions

We actively maintain the latest version of **Periodic Table Explorer**.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅ Yes     |
| Older Versions | ❌ No      |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report

* Email: **[your-email@example.com](mailto:your-email@example.com)**
* Or open a **private security advisory** on GitHub (preferred)

⚠️ Please **do not create a public issue** for security vulnerabilities.

---

## 🛡️ What to Include

When reporting a vulnerability, include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Screenshots or proof-of-concept (if applicable)
* Suggested fix (optional but appreciated)

---

## ⏱️ Response Time

We aim to:

* Acknowledge your report within **48 hours**
* Provide an update within **3–5 working days**
* Release a fix as soon as possible

---

## 🔒 Security Best Practices

While contributing, please:

* Avoid adding sensitive data (API keys, credentials) in code
* Use environment variables for configuration
* Validate and sanitize user inputs
* Keep dependencies up to date
* Follow secure coding practices

---

## 🙌 Responsible Disclosure

We appreciate responsible disclosure and will give proper credit to reporters (if desired).

Thank you for helping make **Periodic Table Explorer** safer! 💙
